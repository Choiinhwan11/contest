### 6. **API 명세서**

자연재해 프로젝트에서 사용될 주요 API에 대한 구체적인 명세서입니다. 각 API의 주요 기능, 사용 방법, 엔드포인트, 요청 및 응답 형식을 상세히 설명합니다.

### **1. 로그인 관련 API**

### 1.1. **카카오 로그인 API**

- **엔드포인트**: `https://kauth.kakao.com/oauth/authorize`
- **메서드**: `GET`
- **기능**: 카카오 계정을 통해 사용자가 간편하게 로그인할 수 있도록 인증 토큰을 발급.
- **요청 파라미터**:
    - `client_id`: 카카오 개발자 콘솔에서 발급받은 앱 키
    - `redirect_uri`: 인증 후 리디렉션할 URI
    - `response_type`: 토큰을 받을 방법 (`code`로 설정)
- **응답 형식**:
    - `code`: 인증 코드, 이후 이 코드를 사용하여 액세스 토큰을 요청

### 1.2. **네이버 로그인 API**

- **엔드포인트**: `https://nid.naver.com/oauth2.0/authorize`
- **메서드**: `GET`
- **기능**: 네이버 계정을 통해 간편 로그인.
- **요청 파라미터**:
    - `client_id`: 네이버 개발자 콘솔에서 발급받은 클라이언트 ID
    - `response_type`: `code`로 설정하여 인증 코드 획득
    - `redirect_uri`: 인증 후 리디렉션할 URI
    - `state`: CSRF 공격 방지를 위한 임의의 문자열
- **응답 형식**:
    - `code`: 인증 코드, 이후 이 코드를 사용해 액세스 토큰을 요청
    - `state`: 요청 시 전달한 상태값 그대로 반환

### 1.3. **구글 로그인 API**

- **엔드포인트**: `https://accounts.google.com/o/oauth2/auth`
- **메서드**: `GET`
- **기능**: 구글 계정을 사용한 간편 로그인.
- **요청 파라미터**:
    - `client_id`: 구글 클라우드 플랫폼에서 발급받은 클라이언트 ID
    - `redirect_uri`: 인증 후 리디렉션할 URI
    - `response_type`: `code`로 설정하여 인증 코드 획득
    - `scope`: 사용자 정보에 접근하기 위한 권한 요청 (예: `email`, `profile`)
    - `state`: 요청 시 임의로 생성한 상태값
- **응답 형식**:
    - `code`: 인증 코드, 이후 액세스 토큰 요청에 사용
    - `state`: 요청 시 전달한 상태값 그대로 반환

### **2. 지도 및 위치 관련 API**

### 2.1. **Google Maps API**

- **엔드포인트**: `https://maps.googleapis.com/maps/api/js`
- **메서드**: `GET`
- **기능**: 지도 표시, 마커 추가, 현재 위치 기반 지도 표시 등 다양한 지도 기능을 제공.
- **요청 파라미터**:
    - `key`: Google Cloud Platform에서 발급받은 API 키
    - `callback`: 지도 로드 후 호출할 함수명
    - `libraries`: 추가 기능 로드 (예: `places` 등)
- **응답 형식**: JavaScript 라이브러리 형태로 반환되어 웹페이지에 삽입

### 2.2. **Geolocation API**

- **엔드포인트**: 브라우저 내장 API (`navigator.geolocation.getCurrentPosition`)
- **메서드**: N/A (브라우저 내장 메서드)
- **기능**: 사용자의 현재 위치 정보를 가져옴.
- **요청 파라미터**:
    - `successCallback`: 위치 정보를 가져오는 데 성공했을 때 호출될 콜백 함수
    - `errorCallback`: 위치 정보를 가져오는 데 실패했을 때 호출될 콜백 함수
    - `options`: 위치 정확도, 타임아웃 등 옵션 설정
- **응답 형식**:
    - `coords.latitude`: 위도
    - `coords.longitude`: 경도
    - `coords.accuracy`: 위치의 정확도

### **3. 캘린더 관련 API**

### 3.1. **네이버 캘린더 API**

- **엔드포인트**: `https://apis.naver.com/naver_openapi/v1/calendar`
- **메서드**: `POST`, `GET`, `PUT`, `DELETE`
- **기능**: 네이버 캘린더에 일정 추가, 수정, 삭제 및 조회 기능을 제공.
- **요청 파라미터** (일정 추가 예시):
    - `title`: 일정 제목
    - `calendarId`: 캘린더 ID
    - `startTime`: 일정 시작 시간 (ISO 8601 형식)
    - `endTime`: 일정 종료 시간 (ISO 8601 형식)
    - `description`: 일정 설명
    - `location`: 일정 장소
- **응답 형식**:
    - 일정 추가 시, 생성된 일정의 `eventId` 반환
    - 조회 시, 일정 목록 및 각 일정의 상세 정보 반환

### 3.2. **Google Calendar API**

- **엔드포인트**: `https://www.googleapis.com/calendar/v3/calendars`
- **메서드**: `POST`, `GET`, `PUT`, `DELETE`
- **기능**: Google 캘린더에 일정 추가, 수정, 삭제 및 조회 기능을 제공.
- **요청 파라미터** (일정 추가 예시):
    - `calendarId`: 캘린더 ID (`primary` 사용 가능)
    - `summary`: 일정 제목
    - `location`: 일정 장소
    - `description`: 일정 설명
    - `start`: 일정 시작 시간 (`dateTime` 형식, 타임존 포함)
    - `end`: 일정 종료 시간 (`dateTime` 형식, 타임존 포함)
- **응답 형식**:
    - 일정 추가 시, 생성된 일정의 `id` 반환
    - 조회 시, 일정 목록 및 각 일정의 상세 정보 반환

### **4. 기상 및 재난 관련 API**

### 4.1. **기상청 현재 날씨 API**

- **엔드포인트**: `http://apis.data.go.kr/1360000/VilageFcstInfoService/getUltraSrtNcst`
- **메서드**: `GET`
- **기능**: 기상청의 초단기 실황 정보를 제공. 사용자의 현재 위치 또는 지정된 지역의 실시간 날씨 데이터를 가져옴.
- **요청 파라미터**:
    - `serviceKey`: 공공데이터포털에서 발급받은 서비스 키
    - `numOfRows`: 한 번의 호출에 가져올 데이터의 개수
    - `pageNo`: 페이지 번호
    - `dataType`: 반환할 데이터의 형식 (`JSON` 또는 `XML`)
    - `base_date`: 기준 날짜 (YYYYMMDD 형식)
    - `base_time`: 기준 시간 (HHMM 형식)
    - `nx`: 격자 X 좌표
    - `ny`: 격자 Y 좌표
- **응답 형식**:
    - `baseDate`: 발표 일자
    - `baseTime`: 발표 시각
    - `category`: 자료 구분 코드 (예: `T1H`는 기온, `RN1`은 강수량)
    - `obsrValue`: 자료 값
